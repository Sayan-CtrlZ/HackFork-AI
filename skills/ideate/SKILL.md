## Protocol

You are in **Builder Mode**. Your job is to pressure-test the user's idea through 5 rapid questions, then crystallize it into PROJECT_BRIEF.md.

### Phase 1 — Socratic Rapid Fire (5 questions, one at a time)

Ask these questions sequentially. Do not ask all at once.

1. **Who specifically** is the user? Not "developers" — give me a name, a job, a pain. (Alex, a solo dev who ships at 2 AM)
2. **What is the demo moment?** Describe the single action a judge watches that makes them say "oh wow."
3. **What is explicitly OUT of scope** for this hackathon? Name 3 things you will not build.
4. **What already exists** that's closest to this? What's your one-sentence differentiation?
5. **Which judging criterion** is this project strongest on? Weakest on?

### Phase 2 — Rubric Pre-Check

Invoke the `rubric-scorer` tool with a preliminary assessment based on the idea as described. Show the preliminary score table:

```
┌─────────────────────┬────────┬───────┐
│ Criterion           │ Weight │ Est.  │
├─────────────────────┼────────┼───────┤
│ Agent Quality       │  30%   │  ??   │
│ Skill Design        │  25%   │  ??   │
│ Working Demo        │  25%   │  ??   │
│ Creativity          │  20%   │  ??   │
└─────────────────────┴────────┴───────┘
```

Flag any criterion estimated below 15/max as a risk.

### Phase 3 — Write PROJECT_BRIEF.md

Write the file following the output schema exactly. Do not skip any section.

At the top of the file, include:
```
> Generated by HackFork ideate skill
> Mode: BUILDER
> Rubric pre-score: [total]/100 (preliminary)
```

### Phase 4 — Completion

Invoke `on_skill_complete` hook behavior:
- Log to memory/MEMORY.md
- Print completion summary
- Suggest: "Run `scaffold` next to generate your project structure."
