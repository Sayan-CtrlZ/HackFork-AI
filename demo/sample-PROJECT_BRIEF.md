## Project Name
**StandupBot** — The Git-Aware Standup Generator

## One-Line Pitch
Watches your commits from the last 24 hours and writes your standup update for you — in your team's voice.

## Target User
**Marco**, a senior dev at a 12-person startup who does async standups in Slack. Marco makes 8–20 commits a day and spends 5 minutes every morning turning them into standup bullet points — translating technical commit messages ("fix: race condition in auth middleware") into team-readable updates ("Fixed a login bug that was causing intermittent failures").

## Core Problem
Standup messages are written from memory at 9 AM when commits were made at 2 AM. They're either too vague ("worked on stuff"), too technical ("refactored the BaseServiceImpl"), or forgotten entirely. The information is already in git — nobody is reading it.

## Solution
A CLI agent that runs `git log --since=24h` on your repo, passes the commits to an LLM with your team's voice profile, and writes a ready-to-post Slack standup message in plain English.

## Key Differentiator
Every "standup tool" asks you to fill out a form. StandupBot reads your git history — the actual record of what you did — and writes it for you. Zero input, real output.

## The Demo Moment
Run `standupbot --repo .` in any git repo. In 8 seconds, a ready-to-paste Slack message appears that accurately summarizes the last 24 hours of commits in plain conversational English.

## Scope Boundaries

**IN:**
- CLI tool: `standupbot --repo <path> --since 24h`
- Reads git log, generates standup via LLM
- Outputs to stdout + copies to clipboard

**OUT:**
- ~~Slack API integration (just paste it manually)~~
- ~~Team voice training (use a default voice template)~~
- ~~GUI, web app, or dashboard~~

## Risk Flags
- ✅ No external API other than LLM (no Slack API to break)
- ⚠️ Requires OPENAI_API_KEY or ANTHROPIC_API_KEY in env
