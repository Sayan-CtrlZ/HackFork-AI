## The Solution

**HackFork** is a git-native AI agent with two internal voices:

- 🤝 **The Builder** — Your ruthlessly practical co-founder. Scopes fast, scaffolds faster.
- ⚔️ **The Judge** — The skeptical jury member who finds your fatal weaknesses before a real judge does.

These two voices compose into a 5-skill pipeline that takes you from vague idea to polished pitch:

```
ideate → scaffold → evaluate → pitch → retrospect
```

Every step generates a real file in your git repository. Your project plan, your adversarial score, your demo script — all version-controlled, all grounded in what was actually built.

## Likely Judge Questions & Coached Answers

**Q: "Why would I use this instead of just asking ChatGPT?"**

> HackFork is not a chatbot. It's a stateful pipeline with mandatory steps. The `evaluate` skill enforces an adversarial check — ChatGPT will validate your idea if you ask it to. HackFork's RULES.md says it *must* challenge you, even if you don't ask. It also generates real files into your git repo, so your project brief, score, and pitch are version-controlled artifacts — not chat history you'll lose.

**Q: "The skills are just markdown files. Is this really an agent?"**

> The skills define *behavior protocols* — not prompts. The `evaluate` skill has a mandatory mode switch, reads git history, scores against a rubric, and blocks the `pitch` skill from running without a `JUDGE_REPORT.md`. That's composable, stateful, rule-governed behavior. Adding a new skill is creating one markdown file. Run `gitagent info` to see all 5 discovered immediately.

**Q: "What's the judging rubric you're scoring against?"**

> Exactly the one from this hackathon: Agent Quality (30%), Skill Design (25%), Working Demo (25%), Creativity (20%). It's hardcoded into the `evaluate` skill. HackFork knows the game and plays it explicitly — not implicitly.

**Q: "Show me the adversarial evaluation. What did it say about weaknesses?"**

> Open `JUDGE_REPORT.md` — it's in the repo. Score is 87/100. Three failure modes identified: demo setup friction, no cold-start wow moment, and skills being instruction files rather than runnable code. We've pre-generated all 5 output artifacts to address the first two, and have a 60-second demo of `gitagent info` listing all skills to address the third.

**Q: "Could you actually use this at a real hackathon?"**
